# AGENTS.md

## Project Overview

This is the **os-e** project.

## Coding Standards

- Write clean, readable code with meaningful variable and function names.
- Keep functions small and focused on a single responsibility.
- Prefer composition over inheritance.
- Handle errors explicitly; never silently swallow exceptions.

## Project Structure

- Keep the repository root clean; organize source code into appropriate subdirectories as the project grows.
- Place tests alongside the code they test or in a dedicated `tests/` directory.

## Git Conventions

- Write concise commit messages that explain *why*, not just *what*.
- Use feature branches for new work.

## Dependencies

- Pin dependency versions for reproducibility.
- Prefer well-maintained, widely-used libraries.
